export interface ConversationRow {

**Functions to implement (all throw on error, no {success, error} wrapper):**

1. `generateConversationId(userId1: string, userId2: string): string` -- sort lexicographically, return `${lower}_${higher}`. Pure function.

2. `getOrCreateConversation(currentUserId: string, friendId: string): Promise<ConversationRow>` -- Sort IDs to determine p1/p2. `supabase.from('conversations').upsert({ id: generateConversationId(...), participant1_id: lower, participant2_id: higher }, { onConflict: 'participant1_id,participant2_id' }).select().single()`. Throw if error.

3. `sendMessage(params: { conversationId, senderId, text?, gifUrl?, type? }): Promise<{ messageId: string }>` -- Insert into messages with `conversation_id`, `sender_id`, `type` (default 'text'), `text`, `gif_url`. Select `id`. Throw if error. Log with logger.info.

4. `sendReaction(conversationId: string, senderId: string, targetMessageId: string, emoji: string): Promise<{ messageId: string }>` -- Insert type='reaction' message with `reply_to_id = targetMessageId`, `emoji = emoji`. No text. Return { messageId }.

5. `removeReaction(conversationId: string, senderId: string, targetMessageId: string): Promise<void>` -- Find the reaction message: `supabase.from('messages').select('id').eq('conversation_id', conversationId).eq('sender_id', senderId).eq('reply_to_id', targetMessageId).eq('type', 'reaction').is('unsent_at', null).single()`. Then set `unsent_at = new Date().toISOString()` on it. Throw if error.

6. `sendReply(params: { conversationId, senderId, text, replyToId, replyPreview }): Promise<{ messageId: string }>` -- Insert type='reply' message with `reply_to_id`, `reply_preview` JSONB (shape: `{ sender_id, type, text }`), `text`. Return { messageId }.

7. `getMessages(conversationId: string, userId: string, options?: { cursor?: string; limit?: number }): Promise<MessageRow[]>` -- Query messages WHERE conversation_id = X, ordered by created_at DESC, limit (default 25). If cursor provided, add `.lt('created_at', cursor)`. Filter out deleted-for-me: use a sub-query approach or post-filter. For Supabase REST, use: `.not('id', 'in', subquery)` -- but since Supabase doesn't support subqueries in REST, use a two-step approach: first fetch message_deletions for this user, then filter client-side. Or use an RPC. Simpler approach: fetch messages, then fetch user's message_deletions for those IDs, then filter. Use the simpler approach for now.

8. `markConversationRead(conversationId: string, userId: string): Promise<void>` -- Fetch conversation to determine if user is p1 or p2. Update `last_read_at_p1` or `last_read_at_p2` to `new Date().toISOString()`. Also reset `unread_count_p1` or `unread_count_p2` to 0. Single update query.

9. `softDeleteConversation(conversationId: string, userId: string): Promise<void>` -- Fetch conversation to determine participant position. Update `deleted_at_p1` or `deleted_at_p2` to `new Date().toISOString()`.

10. `unsendMessage(messageId: string): Promise<void>` -- `supabase.from('messages').update({ unsent_at: new Date().toISOString() }).eq('id', messageId)`. RLS ensures only sender can update. Throw if error.

11. `deleteMessageForMe(messageId: string, userId: string): Promise<void>` -- `supabase.from('message_deletions').insert({ message_id: messageId, user_id: userId })`. Throw if error.

12. `sendTaggedPhotoMessage(conversationId: string, senderId: string, taggedPhotoId: string): Promise<{ messageId: string }>` -- Insert type='tagged_photo' message with `tagged_photo_id`. Return { messageId }.

13. `getConversation(conversationId: string): Promise<ConversationRow>` -- `supabase.from('conversations').select('*').eq('id', conversationId).single()`. Throw if error.

Use `logger.info` for successful operations, `logger.error` for caught errors before rethrowing.
