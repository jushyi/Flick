**Part A: Create `src/hooks/useMessages.ts`:**

Replace the Firebase useMessages hook with a PowerSync-based version. Conversations are PowerSync-synced, so the list reads from local SQLite -- zero network latency.

```typescript
import { useState, useEffect, useCallback, useMemo } from 'react';
import { usePowerSync } from '@powersync/react-native';

import { softDeleteConversation, type ConversationRow } from '../services/supabase/messageService';
import { useAuth } from '../context/AuthContext';
import logger from '../utils/logger';
```

The hook should:
1. Get `userProfile` from `useAuth()` for current user ID
2. Use `powersync.watch()` to query conversations where current user is participant1 or participant2:
   ```sql
   SELECT * FROM conversations
   WHERE (participant1_id = ? OR participant2_id = ?)
   AND last_message_at IS NOT NULL
   ORDER BY last_message_at DESC
   ```
3. Filter out soft-deleted conversations: for each row, check if user is p1 or p2, then check if their `deleted_at_p1`/`deleted_at_p2` is set. If set AND `deleted_at` > `last_message_at`, exclude it (soft-deleted and no new messages since)
4. Map each conversation to include a derived `otherUserId` (the participant who is NOT the current user) and `unreadCount` (read from `unread_count_p1` or `unread_count_p2` based on participant position)
5. Provide `deleteConversation(conversationId)` callback that calls `softDeleteConversation`
6. Return `{ conversations, loading, deleteConversation }`

**Important:** The existing Firebase useMessages.js also joins friend profile data. For the Supabase version, the screen can fetch friend profiles separately via useQuery. The hook should NOT duplicate that logic. Just return conversation rows with `otherUserId` derived.

**Parameters:** `useConversation(conversationId: string)`

**Data fetching:**
- Use `useInfiniteQuery` with `queryKeys.conversations.messages(conversationId)`
- `queryFn`: call `getMessages(conversationId, userId, { cursor: pageParam, limit: 25 })`
- `getNextPageParam`: if page has < 25 results, return undefined; otherwise return `lastPage[lastPage.length - 1].created_at`
- `initialPageParam`: `null` (fetch from latest)
- `staleTime`: 30_000 (30 seconds, project convention)

**Realtime subscription:**
- In useEffect, subscribe to Supabase Realtime channel `messages:${conversationId}`:
  - Listen for `postgres_changes` INSERT on `messages` table filtered by `conversation_id=eq.${conversationId}`
  - Listen for `postgres_changes` UPDATE on `messages` table filtered by `conversation_id=eq.${conversationId}` (for unsend)
  - On both events: `queryClient.invalidateQueries({ queryKey: queryKeys.conversations.messages(conversationId) })`
- Cleanup: `return () => { supabase.removeChannel(channel); }` -- prevents Realtime channel leak

**Deduplication:**
- In useMemo, flatten all pages and deduplicate by message ID using a Map (keyed by `message.id`). This prevents duplicate messages when Realtime invalidation refetches pages and cursor boundaries shift.

**Read receipts:**
- On mount and when app returns to foreground (AppState change to 'active'), call `markConversationRead(conversationId, userId)`
- Use `useRef` for AppState listener to avoid stale closure
- Privacy toggle: read user's `read_receipts_enabled` preference from userProfile. If false, skip the write. (The existing Firebase hook reads this from userProfile.settings?.readReceipts)

**Message actions (all as useMutation or plain callbacks):**
1. `handleSendMessage(text: string, gifUrl?: string)` -- calls `sendMessage({ conversationId, senderId: userId, text, gifUrl })`
2. `handleSendReaction(targetMessageId: string, emoji: string)` -- calls `sendReaction`
3. `handleRemoveReaction(targetMessageId: string)` -- calls `removeReaction`
4. `handleSendReply(text: string, replyToId: string, replyPreview: {...})` -- calls `sendReply`
5. `handleUnsendMessage(messageId: string)` -- calls `unsendMessage`
6. `handleDeleteMessage(messageId: string)` -- calls `deleteMessageForMe`
7. `handleSendSnap(localUri: string, caption?: string)` -- calls `uploadAndSendSnap`
8. `handleMarkSnapViewed(messageId: string)` -- calls `markSnapViewed`
9. `handleGetSnapUrl(storagePath: string)` -- calls `getSignedSnapUrl`
10. `handleSendTaggedPhoto(taggedPhotoId: string)` -- calls `sendTaggedPhotoMessage`

Each mutation should invalidate `queryKeys.conversations.messages(conversationId)` on success.

**Return value:**
