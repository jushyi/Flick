**Part A: Create `src/services/supabase/snapService.ts`:**

Import `ImageManipulator` from `expo-image-manipulator`, `FileSystem` from `expo-file-system/legacy`, `decode` from `base64-arraybuffer`, `supabase` from `@/lib/supabase`, `sendMessage` from `./messageService`, `logger` from `../../utils/logger`.

Constants: `MAX_RETRIES = 3`, `BACKOFF_DELAYS = [1000, 2000, 4000]`.

1. `uploadAndSendSnap(conversationId: string, senderId: string, localUri: string, caption?: string | null): Promise<{ messageId: string }>`:
   - Wrap in retry loop (up to MAX_RETRIES with exponential backoff)
   - Compress: `ImageManipulator.manipulateAsync(localUri, [{ resize: { width: 1080 } }], { compress: 0.9, format: ImageManipulator.SaveFormat.WEBP })`
   - Read as base64: `FileSystem.readAsStringAsync(compressed.uri, { encoding: FileSystem.EncodingType.Base64 })`
   - Decode: `decode(base64)`
   - Generate storage path: `${senderId}/${Date.now()}-${Math.random().toString(36).substring(2, 9)}.webp`
   - Upload: `supabase.storage.from('snaps').upload(storagePath, arrayBuffer, { contentType: 'image/webp', cacheControl: 'no-store' })`
   - Send message: `sendMessage({ conversationId, senderId, type: 'snap', text: caption ? caption.substring(0, 150) : null })`
   - Update message with snap_storage_path: `supabase.from('messages').update({ snap_storage_path: storagePath }).eq('id', result.messageId)`
   - Return `{ messageId }`
   - On retry failure, await `BACKOFF_DELAYS[attempt - 1]` ms
   - On final failure, throw

2. `markSnapViewed(messageId: string): Promise<void>`:
   - `supabase.from('messages').update({ snap_viewed_at: new Date().toISOString() }).eq('id', messageId)`
   - Throw if error
   - Note: this triggers the PG `handle_snap_viewed` trigger which calls the snap-cleanup Edge Function

3. `getSignedSnapUrl(snapStoragePath: string): Promise<string>`:
   - `supabase.storage.from('snaps').createSignedUrl(snapStoragePath, 300)` (300 seconds = 5 minutes)
   - Throw if error
   - Return `data.signedUrl`

Types:

```typescript
export type StreakState = 'default' | 'building' | 'pending' | 'active' | 'warning';

export interface StreakData {
  id: string;
  user1_id: string;
  user2_id: string;
  day_count: number;
  last_snap_at_user1: string | null;
  last_snap_at_user2: string | null;
  last_mutual_at: string | null;
  expires_at: string | null;
  warning_sent: number; // SQLite boolean (0/1)
}
```

1. `generateStreakId(userId1: string, userId2: string): string` -- sort, return `${lower}_${higher}`

2. `deriveStreakState(streak: StreakData | null, currentUserId: string): StreakState`:
   - `null` -> 'default'
   - No `expires_at` (not yet activated):
     - Determine isUser1 = (streak.user1_id === currentUserId)
     - mySnap = isUser1 ? last_snap_at_user1 : last_snap_at_user2
     - theirSnap = isUser1 ? last_snap_at_user2 : last_snap_at_user1
     - mySnap && !theirSnap -> 'pending'
     - mySnap || theirSnap -> 'building'
     - else -> 'default'
   - `warning_sent === 1` (truthy in SQLite) -> 'warning'
   - `day_count >= 3` -> 'active'
   - Else: check mySnap/theirSnap for 'pending' / 'building'

3. `getStreakColor(streakState: StreakState, dayCount: number): string`:
   - 'warning' -> streak warning color
   - 'active' + dayCount >= 50 -> tier3 color
   - 'active' + dayCount >= 10 -> tier2 color
   - 'active' -> tier1 color
   - 'pending' -> pending color
   - 'building' -> building color
   - default -> default color
   - Read `src/constants/colors.js` to find exact color key paths. If `colors.streak` doesn't exist, define inline hex colors matching the existing Firebase streakService.js values.
