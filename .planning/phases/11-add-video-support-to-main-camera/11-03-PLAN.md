Create `src/components/RecordingProgressRing.js` — an animated circular progress ring that fills clockwise during video recording.

**Props:**
- `isRecording` (boolean) — when true, starts filling from 0 to 100% over `maxDuration` seconds
- `maxDuration` (number, default 30) — maximum recording duration in seconds
- `size` (number, default 80) — outer diameter of the ring (should match or slightly exceed the shutter button outer size)
- `strokeWidth` (number, default 4) — ring thickness
- `color` (string, default `colors.accent.red` or a recording-red color) — ring fill color

**Implementation:**
1. Use `react-native-svg` Circle component with `stroke-dasharray` and `stroke-dashoffset` technique.
2. Use `react-native-reanimated` `useSharedValue` + `useAnimatedProps` for smooth UI-thread animation.
3. When `isRecording` transitions to true:
   - Reset progress shared value to 0
   - Animate to 1 over `maxDuration * 1000` ms using `withTiming` with linear easing
4. When `isRecording` transitions to false:
   - Cancel the animation (the shared value stays where it stopped)
5. The ring should be positioned absolutely, centered over the shutter button. The parent (CameraScreen) will handle positioning.
6. Ring fills clockwise starting from the top (12 o'clock position). Use `rotation: -90deg` transform on the SVG to start from top.

**Visual style:**
- Background ring: same circle but with very low opacity (0.2) to show the track
- Foreground ring: solid color that fills
- Use `colors.accent.red` or a bold red (#FF3B30) for the recording indicator — per Claude's discretion on progress ring color

**Import organization:** React core, then react-native, then reanimated, then svg, then constants.

  <action>
Create `src/components/VideoPlayer.js` — a reusable video player component wrapping expo-video's VideoView with custom controls.

**Props:**
- `source` (string) — video URL to play
- `isMuted` (boolean) — whether audio is muted
- `onToggleMute` (function) — callback when mute toggle is tapped
- `loop` (boolean, default true) — whether to loop the video
- `autoPlay` (boolean, default true) — whether to auto-play when mounted
- `showControls` (boolean, default true) — whether to show progress bar and mute toggle
- `onPlayToEnd` (function, optional) — callback when video finishes playing (used for stories auto-advance)
- `onTimeUpdate` (function, optional) — callback with { currentTime, duration } on each time update
- `style` (ViewStyle) — container style
- `contentFit` (string, default 'cover') — how to fit video in container
- `isVisible` (boolean, default true) — controls play/pause based on viewport visibility

**Implementation:**
1. Use `useVideoPlayer` from `expo-video` to create a player instance:
   ```javascript
   import { VideoView, useVideoPlayer } from 'expo-video';
   import { useEvent } from 'expo';
   ```
2. Configure player with loop, muted settings in the setup callback.
3. Listen for `playToEnd` event via `useEvent(player, 'playToEnd', ...)` — call `onPlayToEnd` if provided.
4. Listen for status updates to track currentTime and duration for progress bar.
5. When `isVisible` changes: call `player.play()` or `player.pause()`.
6. When `isMuted` changes: update `player.muted`.
7. Render `VideoView` with `nativeControls={false}` and `contentFit`.

**Controls overlay (when showControls=true):**
- Bottom bar with:
  - Progress bar: thin horizontal bar showing currentTime/duration progress. Use the same style as the existing stories progress bar (thin, white, ~2px height). Use a View with percentage width based on currentTime/duration.
  - Mute toggle button: small speaker icon (use PixelIcon if a speaker icon exists, otherwise use a simple Text toggle "M"/"U"). Positioned in bottom-right corner.
- Controls should be semi-transparent overlay.

**Cleanup:**
- Call `player.release()` or equivalent cleanup when component unmounts (if needed — check expo-video docs).

**Import organization:**
1. React core
2. React Native
3. expo-video, expo
4. Components (PixelIcon)
5. Constants (colors)
