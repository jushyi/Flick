  <name>Task 1: Add hold-to-record logic to useCameraBase</name>
  <files>src/hooks/useCameraBase.js</files>
  <action>
Extend `useCameraBase.js` with video recording state and hold-to-record gesture handling:

1. **Add new state and refs:**
   ```javascript
   const [isRecording, setIsRecording] = useState(false);
   const [cameraMode, setCameraMode] = useState('picture'); // 'picture' | 'video'
   const [recordingDuration, setRecordingDuration] = useState(0);
   const holdTimerRef = useRef(null);
   const recordingTimerRef = useRef(null); // interval for duration tracking
   const isFacingLockedRef = useRef(false); // lock facing during recording
   ```

2. **Add constants:**
   ```javascript
   const HOLD_THRESHOLD_MS = 500;
   const MAX_RECORDING_DURATION = 30; // seconds
   ```
   Export these constants for CameraScreen to use.

3. **Create `handlePressIn` handler:**
   - Call `lightImpact()` for immediate tactile feedback (existing behavior).
   - Start a setTimeout at `HOLD_THRESHOLD_MS`. If the timeout fires:
     a. Switch `cameraMode` to `'video'`
     b. Wait 100ms for camera reconfiguration (the ~500ms hold threshold + 100ms buffer absorbs iOS reconfiguration delay)
     c. Call `lightImpact()` for recording-start haptic
     d. Set `isRecording = true`, lock facing (`isFacingLockedRef.current = true`)
     e. Start `recordingTimerRef` interval (every 1 second, increment `recordingDuration`)
     f. Call `cameraRef.current.recordAsync({ maxDuration: MAX_RECORDING_DURATION })` — this returns a promise that resolves with `{ uri }` when recording stops
     g. When recordAsync resolves (either user stops or maxDuration reached), call `handleRecordingComplete(result)`

4. **Create `handlePressOut` handler:**
   - Clear the hold timer.
   - If `isRecording`:
     a. Call `lightImpact()` for recording-stop haptic
     b. Call `cameraRef.current.stopRecording()` — this causes the `recordAsync` promise to resolve
     c. Set `isRecording = false`, unlock facing, reset `recordingDuration`, clear `recordingTimerRef`
     d. Switch `cameraMode` back to `'picture'`
   - If NOT recording (the hold timer hadn't fired = it was a tap):
     a. If in snap mode, call the existing `takePicture()` which invokes `onSnapCapture({ uri, mediaType: 'photo' })`
     b. If in normal mode, call the existing `takePicture()` function

5. **Create `handleRecordingComplete` function:**
   - Receives `result` from `recordAsync` containing `{ uri }`
   - If in snap mode: call `onSnapCapture({ uri: result.uri, mediaType: 'video' })`. The `onSnapCapture` callback is provided by CameraScreen (see Task 2 step 4) and navigates to SnapPreviewScreen with `mediaType` forwarded. Always record audio per locked decision.
   - If in normal mode:
     a. Calculate duration from `recordingDuration` state (or use a stopwatch ref)
     b. Call `addToQueue(user.uid, result.uri, 'video', recordingDuration)` (using updated addToQueue from Plan 02)
     c. Play the card capture animation (same as photo)
     d. Optimistically increment darkroom counts

6. **Modify `toggleCameraFacing`** (in platform hooks or base):
   - If `isFacingLockedRef.current === true`, return early (do nothing). This locks facing during recording per user decision (expo-camera stops recording when flipping).

7. **Update the return object** to include:
   - `isRecording`, `cameraMode`, `recordingDuration`
   - `handlePressIn`, `handlePressOut`
   - `MAX_RECORDING_DURATION` (for progress ring)

8. **Cleanup:** In a useEffect cleanup, clear `holdTimerRef` and `recordingTimerRef` if they exist.

**Important:** The existing `takePicture` function remains unchanged. `handlePressOut` calls it when the press was a tap (not a hold). The existing `onPressIn`/`onPressOut` on CameraScreen's Pressable will be replaced with these new handlers.
