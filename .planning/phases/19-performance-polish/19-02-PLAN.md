    - Test: getSignedUrlExpiry returns expiry timestamp in ms from a valid signed URL JWT

    - Test: isUrlNearExpiry returns true when URL has no parseable expiry (safety fallback)
    - Test: FEED_CARD_WIDTH equals 400
  </behavior>
  <action>
    Create `src/utils/imageUrl.ts` with the following exports:

    1. `FEED_CARD_WIDTH = 400` constant

    2. `getTransformedPhotoUrl(storagePath: string, options?: { width?: number; quality?: number; bucket?: string }): string`
       - Default bucket: 'photos'
       - If no width provided: call `supabase.storage.from(bucket).getPublicUrl(storagePath)` and return `data.publicUrl`
       - If width provided: call `supabase.storage.from(bucket).getPublicUrl(storagePath, { transform: { width, ...(quality ? { quality } : {}) } })` and return `data.publicUrl`

    3. `getSignedUrlExpiry(signedUrl: string): number | null`
       - Parse the URL, extract `token` query parameter
       - If no token, return null
       - Decode JWT payload (second segment): use `Buffer.from(token.split('.')[1], 'base64').toString()` instead of `atob()` for Android/Hermes compatibility (per RESEARCH.md Pitfall 5)
       - Import Buffer from 'buffer' package (transitive dependency, already available)
       - Parse JSON payload, return `payload.exp * 1000` (convert seconds to ms)
       - Wrap in try/catch, return null on any error

    4. `isUrlNearExpiry(signedUrl: string, thresholdMs = 60_000): boolean`
       - Call getSignedUrlExpiry, if null return true (can't determine = treat as expired)
       - Return `Date.now() + thresholdMs >= expiry`

    Create `__tests__/utils/imageUrl.test.ts`:
       - Mock `@/lib/supabase` with `supabase.storage.from().getPublicUrl()` returning predictable URLs
       - Test all behaviors listed above
       - For signed URL tests, construct a real JWT-like token: `header.payload.signature` where payload is base64-encoded JSON with `exp` claim
       - Use `jest.spyOn(Date, 'now')` to control time for expiry tests
  </action>
