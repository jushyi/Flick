       - Read the screen to find where snap images are rendered (look for Image/expo-image components displaying snap URLs)
       - Create a small `useSnapUrl` pattern (can be inline or a local hook within the file):
         ```
         function useSnapUrl(originalUrl: string, storagePath: string) {
           const [resolvedUrl, setResolvedUrl] = useState(originalUrl);
           useEffect(() => {
             if (originalUrl && isUrlNearExpiry(originalUrl)) {
               refreshSignedUrlIfExpiring(originalUrl, storagePath)
                 .then(newUrl => { if (newUrl !== originalUrl) setResolvedUrl(newUrl); });
             } else {
               setResolvedUrl(originalUrl);
             }
           }, [originalUrl, storagePath]);
           return resolvedUrl;
         }
         ```
       - Use `resolvedUrl` as the image source instead of the raw snap URL
       - Import `{ isUrlNearExpiry }` from `@/utils/imageUrl`
       - Import `{ refreshSignedUrlIfExpiring }` from signedUrlService

    **Important notes:**
    - expo-image with `cachePolicy="memory-disk"` will serve the cached image while the URL refreshes, so no visible flash occurs
    - Only snap images (signed URLs) need this -- public photo URLs are permanent CDN URLs and never expire
    - The 60-second threshold in isUrlNearExpiry gives enough time to refresh before actual expiry
    - The useSnapUrl pattern prevents re-render cascades per review concern: it only triggers refresh once per originalUrl change, and the state update is a no-op if the URL didn't actually change
