  <action>
    **Per D-10/PERF-05: Snap images must never flash expired URLs. Check expiry before rendering and trigger background refresh.**

    1. **signedUrlService.ts**: Add a `refreshIfExpiring` helper function:
       - Import `{ isUrlNearExpiry }` from `@/utils/imageUrl`
       - `export async function refreshSignedUrlIfExpiring(currentUrl: string, storagePath: string, bucket?: string): Promise<string>`
       - If `isUrlNearExpiry(currentUrl)` returns false, return `currentUrl` unchanged
       - If near expiry, generate a new signed URL using the existing `createSignedUrl` function in this service
       - Return the new URL
       - Wrap in try/catch: on error, return currentUrl (graceful degradation -- serve potentially expired URL rather than crash)
       - Use logger.warn on refresh failure

    2. **ConversationScreen.js** (or the snap viewer component):
       - Read the screen to find where snap images are rendered (look for Image/expo-image components displaying snap URLs)
       - Before rendering a snap image, add a check:
         - Import `{ isUrlNearExpiry }` from `@/utils/imageUrl`
         - Import `{ refreshSignedUrlIfExpiring }` from the signed URL service
         - When a snap message with an image URL is about to render, check `isUrlNearExpiry(snapUrl)`
         - If near expiry, trigger `refreshSignedUrlIfExpiring()` in a background effect (useEffect or inline async call)
         - While refreshing, continue serving the current (cached by expo-image) URL -- do NOT show a loading state
         - Once the new URL arrives, update the image source (via local state or query cache update)
       - The pattern should be:
         ```
         const [resolvedUrl, setResolvedUrl] = useState(originalUrl);
         useEffect(() => {
           if (isUrlNearExpiry(originalUrl)) {
             refreshSignedUrlIfExpiring(originalUrl, storagePath).then(setResolvedUrl);
           }
         }, [originalUrl]);
         // Render <Image source={{ uri: resolvedUrl }} />
         ```
       - Alternatively, if snaps are rendered via a SnapImage component or similar, modify that component instead

    **Important notes:**
    - expo-image with `cachePolicy="memory-disk"` will serve the cached image while the URL refreshes, so no visible flash occurs
    - Only snap images (signed URLs) need this -- public photo URLs are permanent CDN URLs and never expire
    - The 60-second threshold in isUrlNearExpiry gives enough time to refresh before actual expiry
