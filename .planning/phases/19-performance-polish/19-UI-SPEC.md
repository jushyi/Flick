## Interaction Contracts

### Skeleton Screen Transitions

| State | What Shows |
|-------|-----------|
| `isLoading && !data` | Skeleton screen (first load, no cache) |
| `isLoading && data` | Real data (stale-while-revalidate, background refetch) |
| `!isLoading && data.length === 0` | EmptyState component |
| `!isLoading && data.length > 0` | Real content |

No crossfade or animation between skeleton and real content. Instant swap (matches existing FeedLoadingSkeleton behavior).

### Optimistic Update Flow

1. User taps action (react, comment, mark-read, block, album edit)
2. UI updates IMMEDIATELY via `queryClient.setQueryData`
3. Mutation fires to server in background
4. **On success:** `onSettled` invalidates query, server data replaces optimistic data (usually identical)
5. **On error:** `onError` restores snapshot from `onMutate` context, shows toast with error message
6. No loading indicators during optimistic updates. The action appears instant.

### Image Loading States

| Context | Image Size | Source |
|---------|-----------|--------|
| Feed card | 400px width, WebP | `getPublicUrl` with `transform: { width: 400 }` |
| PhotoDetail | Original (1080px) | `getPublicUrl` without transform |
| Story card thumbnail | 400px width, WebP | `getPublicUrl` with `transform: { width: 400 }` |
| Profile photo | 400px width, WebP | `getPublicUrl` with `transform: { width: 400 }` |
| Snap image | Original | Signed URL (5-min expiry, proactive refresh at 1-min threshold) |

expo-image handles all image display with `cachePolicy="memory-disk"`. No custom loading placeholders -- expo-image shows the cached version while loading the new size.

### Prefetch Strategy

| Trigger | What to Prefetch | Size |
|---------|-----------------|------|
| Feed loads | First image of each friend's story | 400px (card size) |
| Story viewing starts | Next 3 photos in current friend's story | Full-res (1080px) |
| Story friend transition | First 3 photos of next friend | Full-res (1080px) |

Use `Image.prefetch(urls, 'memory-disk')` from expo-image (existing pattern from useDarkroom).
