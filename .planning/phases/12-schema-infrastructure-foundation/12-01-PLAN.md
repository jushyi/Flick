  </read_first>
  <action>
1. Update `tsconfig.json` to:
   ```json
   {
     "extends": "expo/tsconfig.base",
     "compilerOptions": {
       "strict": false,
       "allowJs": true,
       "noEmit": true,
       "baseUrl": ".",
       "paths": {
         "@/*": ["src/*"]
       },
       "resolveJsonModule": true,
       "esModuleInterop": true,
       "skipLibCheck": true
     },
     "include": ["src/**/*", "supabase/**/*.ts", "app.config.js"],
     "exclude": ["node_modules", "functions", "__tests__"]
   }
   ```
   Key decisions:
   - `strict: false` to avoid breaking existing JS files (per research Pitfall 6)
   - `allowJs: true` so existing .js files coexist with new .ts files
   - `paths` with `@/*` maps to `src/*` for clean imports
   - `include` adds `supabase/**/*.ts` for Edge Function type checking
   - `exclude` keeps `functions/` (Firebase Cloud Functions) out of TS checking

2. Create `src/types/database.ts` as a placeholder that will be overwritten by `supabase gen types typescript`:
   ```typescript
   // Auto-generated by: npm run db:types
   // Do not edit manually. Regenerate after schema changes.
   // This placeholder will be replaced once Supabase is linked and schema is deployed.

   export type Json =
     | string
     | number
     | boolean
     | null
     | { [key: string]: Json | undefined }
     | Json[]

   export interface Database {
     public: {
       Tables: Record<string, never>
       Views: Record<string, never>
       Functions: Record<string, never>
       Enums: Record<string, never>
     }
   }
   ```

3. Create `src/lib/supabase.ts`:
   ```typescript
   import 'react-native-url-polyfill/dist/polyfill';
   import { createClient } from '@supabase/supabase-js';
   import AsyncStorage from '@react-native-async-storage/async-storage';
   import { Database } from '@/types/database';

   const supabaseUrl = process.env.EXPO_PUBLIC_SUPABASE_URL ?? '';
   const supabaseAnonKey = process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY ?? '';

   export const supabase = createClient<Database>(supabaseUrl, supabaseAnonKey, {
     auth: {
       storage: AsyncStorage,
       autoRefreshToken: true,
       persistSession: true,
       detectSessionInUrl: false,
     },
   });
   ```

4. Install `react-native-url-polyfill` if not already present:
   ```bash
   npx expo install react-native-url-polyfill
   ```

5. Create `src/lib/` directory if it does not exist.

6. Update `.env.example` (create if not exists) with:
   ```
   EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
   EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
   ```
