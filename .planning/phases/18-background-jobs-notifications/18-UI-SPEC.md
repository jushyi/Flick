# Phase 18 -- UI Design Contract

> Visual and interaction contract for Phase 18: Background Jobs & Notifications.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Classification: Backend-Only

Phase 18 is entirely server-side infrastructure. It ports Firebase Cloud Functions to Supabase pg_cron, Edge Functions, and PostgreSQL triggers. **No new screens, components, or visual elements are introduced.** The only user-facing surface is push notification copy delivered to the OS notification tray.

Client-side changes are limited to:
1. Updating `liveActivityService` to store `push_to_start_token` in Supabase users table (was Firestore)
2. Updating `notificationService` to store `expo_push_token` in Supabase users table (if not already done in Phase 15/16)

Neither change introduces new UI.

## Copywriting Contract

This is the primary deliverable for Phase 18's UI contract. All push notification copy must be ported identically from the existing Cloud Functions. The Edge Function `send-push-notification` must reproduce these exact templates.

### Push Notification Templates

All notifications use the pattern: **title** (sender name or "Flick") + **body** (action description). No emojis in notification body text (per existing project convention).

| Event Type | Title | Body | Randomized | Source |
|------------|-------|------|------------|--------|
| `photo_reveal` (1 photo) | Flick | Your photo is ready to reveal! | No | functions/index.js:495 |
| `photo_reveal` (N photos) | Flick | Your {count} photos are ready to reveal! | No | functions/index.js:496 |
| `friend_request` | {senderName} | sent you a friend request | No | functions/index.js:619 |
| `friend_accept` | {acceptorName} | accepted your friend request | No | functions/index.js:737 |
| `comment` | {commenterName} | commented on your photo: {preview} | No | functions/index.js:1302 |
| `tag` (single) | {taggerName} | One of: "tagged you in a photo" / "added you to their latest snap" | Yes (2 templates) | functions/index.js:32-34 |
| `tag_batch` (N photos) | {taggerName} | One of: "tagged you in {count} photos" / "added you to {count} of their snaps" | Yes (2 templates) | functions/index.js:45-47 |
| `snap` | {senderName} | One of: "sent you a snap" / "just snapped you" / "New snap" | Yes (3 templates) | functions/index.js:56 |
| `pinned_snap` | {senderName} | pinned a snap to your screen | No | functions/index.js:3117 |
| `reaction_batch` | {reactorName} | reacted {emojiSummary} to your photo | No | tasks/sendBatchedNotification.js:247 |
| `streak_warning` | Flick | One of: "Your streak with {name} is about to expire!" / "Don't let your streak with {name} die!" / "Quick -- snap {name} before your streak ends!" | Yes (3 templates) | functions/index.js:87-91 |
| `new_message` (text) | {senderName} | {messageText} (truncated) | No | functions/index.js:3108+ |
| `new_message` (reaction) | {senderName} | Reacted | No | functions/index.js:3033 |
| `new_message` (tagged_photo) | {senderName} | Tagged you in a photo | No | functions/index.js:3120 |
| `system_screenshot` | {senderName} | {text} or "Someone screenshotted a snap" | No | functions/index.js:3035 |
| `deletion_reminder` | Flick | Your account will be permanently deleted in 3 days. Log in to cancel if you've changed your mind. | No | functions/index.js:1985-1986 |

### Randomized Template Rules

- `getRandomTemplate(templates)` picks one template at random from the array per invocation
- Templates use `{name}`, `{count}` as placeholders, replaced at send time
- Reaction summary format: `{emoji}x{count}` pairs joined by spaces (e.g., "heart-eyesx2 firex1")

### Notification Channels (Android)

| Channel ID | Usage |
|------------|-------|
| `default` | All notification types unless overridden |

Channel is passed in `data.channelId` or defaults to `'default'`. No new channels introduced in Phase 18.

### Notification Priority

All notifications sent with `priority: 'high'` to ensure delivery to locked/background devices.

### Notification Data Payload

Each notification includes a `data` object for deep linking. The `type` field determines which screen the app navigates to on tap:

| Type | Deep Link Target |
|------|-----------------|
| `photo_reveal` | Darkroom screen |
| `friend_request` | Activity / Friends screen |
| `friend_accept` | Activity / Friends screen |
| `comment` | PhotoDetail screen (via photoId) |
| `tag` / `tag_batch` | PhotoDetail screen (via photoId) |
| `snap` | Conversation screen |
| `pinned_snap` | Conversation screen (triggers Live Activity) |
| `reaction_batch` | PhotoDetail screen (via photoId) |
| `streak_warning` | Conversation screen |
| `new_message` | Conversation screen (via conversationId) |
| `deletion_reminder` | Profile / Settings screen |
