#!/usr/bin/env npx tsx
/**
 * Reset dev Supabase to clean state for migration re-run.
 * Usage: npx tsx scripts/reset-dev-supabase.ts

6. **Verify clean state**: Query each table to confirm 0 rows.

Per D-15: Create a validation script that checks all migrated data is correct and all services work against Supabase.

Create `scripts/validate-migration.ts`:

```typescript
#!/usr/bin/env npx tsx
/**
 * Validate Firebase-to-Supabase migration completeness.
 * Usage: npx tsx scripts/validate-migration.ts
 *
 * Checks: data integrity, service operations, zero Firebase fallbacks.
 */
```

The script must validate:

1. **Data integrity checks:**
   - Users table has expected row count (compare with Firebase export)
   - Photos table has expected row count
   - All photos have valid storage URLs (spot-check 10 random photos)
   - Friendships are bidirectional (user1_id < user2_id constraint)
   - Conversations have valid participant references
   - No orphaned records (photos without users, messages without conversations)

2. **Service operation checks:**
   - Auth: `supabase.auth.getSession()` returns valid session or null (no error)
   - Storage: Can generate signed URL for a random photo
   - Database: Can query each table via `supabase.from('table').select('id').limit(1)`
   - RPC: Can call key RPCs (get_feed, etc.)

3. **Zero Firebase fallback check:**
   - `grep -r "@react-native-firebase" src/ --include="*.ts" --include="*.tsx"` returns 0 results
   - `grep -r "services/firebase" src/ --include="*.ts" --include="*.tsx"` returns 0 results
   - No Firebase imports in any source file

4. **Output format:**
   ```
   [PASS] Users: 47 rows migrated
   [PASS] Photos: 892 rows migrated
   [PASS] Storage URLs: 10/10 valid
   [FAIL] Conversations: 2 orphaned records found
   ...
   RESULT: 15/16 checks passed, 1 failed
   ```

Exit code 0 if all pass, 1 if any fail.
  </action>
