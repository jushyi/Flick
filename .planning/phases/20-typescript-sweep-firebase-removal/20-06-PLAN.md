**Step 6: Enable strict TypeScript**
Update `tsconfig.json`:
```json
{
  "extends": "expo/tsconfig.base",
  "compilerOptions": {
    "strict": true,
    "allowJs": false,
    "noEmit": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    },
    "resolveJsonModule": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src/**/*", "supabase/**/*.ts", "App.tsx", "app.config.js"],
  "exclude": ["node_modules", "__tests__"]
}
```
Changes: `strict: false` -> `strict: true`, `allowJs: true` -> `allowJs: false`, remove `functions` from exclude (directory deleted), update include to reference `App.tsx` instead of `app.config.js` alone.

Note: Keep `app.config.js` in include (it stays as .js since it's a config file read by Expo, not part of the TS source).

**Step 7: Fix strict mode errors**
Run `npx tsc --noEmit` and fix all errors. Common issues:
- Implicit `any` in callbacks: add explicit parameter types
- Null checks: add `if (x != null)` guards or use optional chaining
- Missing return types on exported functions
- `strictFunctionTypes` violations: ensure callback types match exactly

If error count is overwhelming (500+), consider enabling strict sub-flags incrementally:
1. First: `"strictNullChecks": true` (fix null/undefined issues)
2. Then: `"noImplicitAny": true` (fix implicit any)
3. Then: `"strict": true` (enables all remaining)
