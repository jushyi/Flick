   - SQL: `INSERT INTO photos (id, user_id, status, photo_state, local_uri, image_url, storage_path, thumbnail_data_url, reveal_at, media_type, caption, comment_count, reaction_count, deleted_at, created_at) VALUES (?, ?, 'developing', NULL, ?, NULL, NULL, ?, ?, ?, NULL, 0, 0, NULL, ?)`
   - Params: `[photoId, userId, localUri, thumbnailDataUrl ?? null, revealAt, mediaType ?? 'photo', new Date().toISOString()]`
   - Throws if `getPowerSyncDb()` returns null

2. `updatePhotoAfterUpload(photoId: string, imageUrl: string, storagePath: string): Promise<void>`
   - SQL: `UPDATE photos SET image_url = ?, storage_path = ? WHERE id = ?`
   - Params: `[imageUrl, storagePath, photoId]`
   - CRITICAL: This is called by uploadQueueService after successful upload to fill in the URL

3. `triagePhoto(photoId: string, action: 'journal' | 'archive'): Promise<void>`
   - SQL: `UPDATE photos SET photo_state = ? WHERE id = ?`
   - Params: `[action, photoId]`

4. `batchTriagePhotos(photoIds: string[], action: 'journal' | 'archive'): Promise<void>`
   - Loop over photoIds, execute same UPDATE as triagePhoto for each
   - All via PowerSync local writes

5. `softDeletePhoto(photoId: string): Promise<void>`
   - SQL: `UPDATE photos SET deleted_at = ? WHERE id = ?`
   - Params: `[new Date().toISOString(), photoId]`

6. `restorePhoto(photoId: string): Promise<void>`
   - SQL: `UPDATE photos SET deleted_at = NULL WHERE id = ?`

7. `getUserPhotos(userId: string): Promise<Photo[]>`
   - SQL: `SELECT * FROM photos WHERE user_id = ? AND deleted_at IS NULL ORDER BY created_at DESC`
   - Maps each row through `mapToPhoto()` (snake_case -> camelCase)

8. `getPhotoById(photoId: string): Promise<Photo | null>`
   - SQL: `SELECT * FROM photos WHERE id = ? AND deleted_at IS NULL`
   - Returns null if not found, else maps through `mapToPhoto()`

9. `updatePhotoCaption(photoId: string, caption: string): Promise<void>`
   - SQL: `UPDATE photos SET caption = ? WHERE id = ?`

**Step 3: Create `src/services/supabase/darkroomService.ts`:**

All functions use PowerSync local reads/writes. Throw-on-error pattern.

Exports:

1. `checkAndRevealPhotos(userId: string): Promise<number>`
   - Query: `SELECT id FROM photos WHERE user_id = ? AND status = 'developing' AND reveal_at <= ? AND deleted_at IS NULL`
   - Params: `[userId, new Date().toISOString()]`
   - For each ready photo: `UPDATE photos SET status = 'revealed' WHERE id = ?`
   - Returns count of revealed photos

2. `getNextRevealTime(userId: string): Promise<string | null>`
   - Query: `SELECT MIN(reveal_at) as next_reveal FROM photos WHERE user_id = ? AND status = 'developing' AND deleted_at IS NULL`
   - Returns `result?.next_reveal ?? null`

3. `calculateBatchRevealAt(userId: string): Promise<string>`
   - Query existing developing photos: `SELECT MIN(reveal_at) as batch_reveal_at FROM photos WHERE user_id = ? AND status = 'developing' AND deleted_at IS NULL AND reveal_at > ?`
   - Params: `[userId, new Date().toISOString()]`
   - If existing batch exists, return that timestamp (join the batch)
   - If no existing batch, generate random 0-5 minute future timestamp: `new Date(Date.now() + Math.random() * 5 * 60 * 1000).toISOString()`

4. `getDevelopingPhotos(userId: string): Promise<Photo[]>`
   - Query: `SELECT * FROM photos WHERE user_id = ? AND status = 'developing' AND deleted_at IS NULL ORDER BY created_at DESC`
   - Maps through `mapToPhoto` imported from photoService

5. `getRevealedPhotos(userId: string): Promise<Photo[]>`
   - Query: `SELECT * FROM photos WHERE user_id = ? AND status = 'revealed' AND deleted_at IS NULL ORDER BY created_at DESC`
   - Maps through `mapToPhoto` imported from photoService
