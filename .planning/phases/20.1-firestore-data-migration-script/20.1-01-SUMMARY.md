## Decisions Made
- Users migrated via UPDATE (not INSERT) since auth migration Edge Function already creates user rows with Supabase Auth UUIDs
- Photo reactions and tags extracted from inline photo document data (reactions map, taggedUserIds array) rather than separate Firestore collection reads
- Viewed photos iterated via per-user subcollection reads (users/{uid}/viewedPhotos) since Firestore subcollections require parent traversal
- Friendships sort Supabase UUIDs before insert to satisfy CHECK(user1_id < user2_id) constraint
- Script aborts if no users found in userIdMap -- auth migration must run first
