## Decisions

- **Global sort for self-referencing FKs**: Messages and comments are collected from ALL subcollections, sorted globally by `created_at` ascending, then batch inserted. This ensures `reply_to_id` and `parent_id` references always point to already-inserted rows.
- **Comment likes queue via global**: Used `(global as any).__commentLikeQueue` to pass comment metadata from comments migrator to comment_likes migrator, since comment likes require knowing which photo each comment belongs to for subcollection path traversal.
- **Participant sort tracking**: For conversations and streaks, track whether UUIDs were swapped during CHECK constraint sorting to correctly assign per-participant fields (unread counts, deletion timestamps, snap timestamps).
