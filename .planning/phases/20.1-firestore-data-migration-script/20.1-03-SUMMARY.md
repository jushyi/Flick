## What Was Built

Cross-database verification script, full migration cycle orchestrator, and environment variable templates for the iterative dev workflow.

## Key Files

### Created
- `scripts/verify-migration-cross-db.ts` (571 lines) — Connects to both Firestore and Supabase simultaneously. Compares doc/row counts for all 17 collections (top-level, subcollection, and inline-derived). Samples random users for field-by-field comparison. Supports `--user <firebase_uid>` for per-user deep verification with field diffs, per-user data counts, and sample photo comparison.
- `scripts/run-migration-cycle.ts` (235 lines) — Orchestrates the full migration pipeline: reset -> auth confirmation -> data migration -> storage migration -> verification. Stops on first failure. Has `--skip-*` flags for re-runs and prod safety confirmation.
- `.env.dev.example` — Dev environment variable template
- `.env.prod.example` — Prod environment variable template with safety warnings

_None_

### Existing (unchanged)

## Decisions

- **Inline-derived tables get existence checks only**: photo_reactions, photo_tags, album_photos, message_deletions, and comment_likes are derived from inline data in Firestore docs. Direct count comparison is impractical, so we verify Supabase has rows and report the count.
- **MCP config unchanged**: `.mcp.json` already had `firebase-dev` and `firebase-prod` MCP server entries with `@gannonh/firebase-mcp`, so no modifications needed.
- **User sample matching by created_at window**: For per-user photo comparison, photos are matched by `user_id + created_at` within a 1-second window since IDs change during migration.
