    **Workflow per D-09:**

    1. Claude reads the next pending test case from 21-UAT.md

       c. Claude writes the fix (typically: import the Supabase service function, replace the stub with the real call)
       d. Claude runs `npm test -- --bail` to verify the fix doesn't break tests
       e. Claude commits the fix: `fix(21): wire [function] to [supabase service] for [screen]`
       f. User reloads the app (shake -> reload for JS changes)
       g. User re-tests the specific flow
       h. Claude updates 21-UAT.md with `result: pass` (after retest passes) or continues debugging
