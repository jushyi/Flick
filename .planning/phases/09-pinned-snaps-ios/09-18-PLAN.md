# 09-18: Firebase Admin SDK & Cloud Functions v2 Migration

## Problem

The project uses `firebase-admin@^12` and `firebase-functions@^4.5.0`. Push-to-start Live Activities (09-16) requires `firebase-admin@^13.5+` which has peer dependency on `firebase-functions@^7+`. Additionally, ~15 Cloud Functions still use the deprecated v1 trigger API (`functions.firestore.document(...).onCreate(...)`) which generates console warnings and will be removed in v7.

## Scope

### Package Upgrades
- `firebase-admin`: ^12.0.0 → ^13.5.0+
- `firebase-functions`: ^4.5.0 → ^7.0.0+
- `firebase-functions-test`: ^3.1.0 → latest compatible version

### v1 → v2 Trigger Migration

All Firestore triggers using `functions.firestore.document(path).onCreate/onWrite/onUpdate` must be converted to v2 equivalents:

**v1 pattern (deprecated):**
```javascript
exports.myFunction = functions
  .firestore.document('collection/{docId}')
  .onCreate(async (snap, context) => {
    const data = snap.data();
    const docId = context.params.docId;
  });
```

**v2 pattern:**
```javascript
const { onDocumentCreated } = require('firebase-functions/v2/firestore');

exports.myFunction = onDocumentCreated('collection/{docId}', async (event) => {
  const data = event.data.data();
  const docId = event.params.docId;
});
```

### Key Signature Differences (v1 → v2)

| v1 | v2 |
|----|-----|
| `(snap, context)` | `(event)` |
| `snap.data()` | `event.data.data()` |
| `snap.ref` | `event.data.ref` |
| `context.params.docId` | `event.params.docId` |
| `snap.before.data()` (onUpdate/onWrite) | `event.data.before.data()` |
| `snap.after.data()` (onUpdate/onWrite) | `event.data.after.data()` |
| `functions.pubsub.schedule(...)` | `onSchedule` from `firebase-functions/v2/scheduler` |
| `functions.https.onRequest(...)` | `onRequest` from `firebase-functions/v2/https` |

### firebase-admin v13 Breaking Changes

- `admin.firestore.FieldValue` → import from `firebase-admin/firestore` directly
- `admin.firestore.Timestamp` → same
- `admin.storage().bucket()` → may need explicit bucket name
- Verify `initializeFirestore` with `preferRest: true` still works

## Notes

- This is a **server-side only** change — no native build needed, no OTA needed
- Deploy carefully: all functions re-deploy at once when upgrading the SDK
- The v2 API supports `memory` and `timeoutSeconds` options on triggers (not just onCall)
- Keep the `const functions = require('firebase-functions')` import temporarily for any functions not yet migrated, remove only when all are converted
- `@google-cloud/tasks` and `expo-server-sdk` should remain compatible
