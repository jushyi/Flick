<objective>
Fix the NSE plist blocker by rewriting `plugins/withNSELiveActivities.js` to use the `@bacons/apple-targets` Xcode project API (`withXcodeProjectBeta` + `setBuildSetting`) instead of the incompatible `xcode` npm package.

Purpose: Plan 06 created the Notification Service Extension but `Activity.request()` fails at runtime with "Target does not include NSSupportsLiveActivities plist key." Root cause: the existing plugin uses the `xcode` npm package (v3.0.1) to modify the pbxproj, but `@bacons/apple-targets` uses `@bacons/xcode` (v1.0.0-alpha.32). These are incompatible pbxproj serializers -- the `xcode` package's output corrupts the pbxproj that `@bacons/apple-targets` wrote, invalidating the NSE build settings. The fix uses `withXcodeProjectBeta` which shares the same parser/serializer as `@bacons/apple-targets`, ensuring build settings survive through the build pipeline.

Output: A rewritten `plugins/withNSELiveActivities.js` that correctly injects `INFOPLIST_KEY_NSSupportsLiveActivities=YES` into the FlickNotificationService target build configurations, and an updated `Info.plist` with the key as belt-and-suspenders.
</objective>

       ```javascript
       /**
        * Expo config plugin to ensure NSSupportsLiveActivities is set in the
        * FlickNotificationService extension's build settings.
        *
        * CRITICAL: This plugin MUST use @bacons/apple-targets' own Xcode project API
        * (withXcodeProjectBeta). Do NOT use the 'xcode' npm package — it has an
        * incompatible pbxproj serializer that corrupts build settings written by
        * @bacons/apple-targets. See 09-RESEARCH.md "NSE Plist Fix" for full diagnosis.
        *
        * Strategy: Add INFOPLIST_KEY_NSSupportsLiveActivities=YES as a build setting.
        * When GENERATE_INFOPLIST_FILE=YES (set by @bacons/apple-targets), Xcode merges
        * INFOPLIST_KEY_* build settings into the generated Info.plist at build time.
        * This ensures NSSupportsLiveActivities=true appears in the final compiled plist.
        */
       const { withXcodeProjectBeta } = require('@bacons/apple-targets/build/with-bacons-xcode');

       module.exports = function withNSELiveActivities(config) {
         return withXcodeProjectBeta(config, async (config) => {
           const project = config.modResults;
           const targets = project.rootObject.props.targets;

           let found = false;
           for (const target of targets) {
             // Match by productName (how @bacons/apple-targets identifies targets internally)
             // Also try getDisplayName() as fallback
             const name = target.props?.productName ||
                          (typeof target.getDisplayName === 'function' ? target.getDisplayName() : null);

             if (name === 'FlickNotificationService') {
               // Add NSSupportsLiveActivities to ALL build configurations (Debug + Release)
               // This sets the INFOPLIST_KEY_NSSupportsLiveActivities build setting,
               // which Xcode translates to NSSupportsLiveActivities=true in the generated plist
               target.setBuildSetting('INFOPLIST_KEY_NSSupportsLiveActivities', 'YES');
               found = true;
               console.log('[withNSELiveActivities] Set INFOPLIST_KEY_NSSupportsLiveActivities=YES for', name);
               break;
             }
           }

           if (!found) {
             console.warn(
               '[withNSELiveActivities] Could not find FlickNotificationService target.',
               'Available targets:',
               targets.map(t => t.props?.productName || t.getDisplayName?.() || 'unknown').join(', ')
             );
           }

           return config;
         });
       };
       ```
