<objective>
Fix the two UAT blockers and the related push notification suppression issue:
1. Push-to-start payload sends empty content-state {} but iOS expects { stack: [...] } -- Live Activity never starts when app is killed
2. removeFromStack native function exists in Swift but is not exported from the TypeScript bridge -- viewing a snap crashes with "removeFromStack is not a function"
3. NSE suppression needs explicit sound=nil and badge=0 to be fully silent

Purpose: These are the two blockers preventing pinned snaps from working in production. Without fix 1, Live Activities never start when the app is killed. Without fix 2, viewing a pinned snap cannot remove it from the stack.

Output: Fixed liveActivitySender.js, updated index.ts with removeFromStack export, hardened NSE suppression.

    To:
    ```swift
    if liveActivityStarted {
        // Fully suppress the push notification banner when Live Activity was created.
        // Set all fields explicitly to prevent any residual alert, sound, or badge.
        let silentContent = UNMutableNotificationContent()
        silentContent.sound = nil
        silentContent.badge = 0
        silentContent.title = ""
        silentContent.body = ""
        silentContent.subtitle = ""
        self.diag("suppressing_notification", ["reason": "live_activity_started"])
        contentHandler(silentContent)
    } else {
        self.diag("showing_notification", ["reason": "live_activity_failed"])
        contentHandler(bestAttemptContent)
    }
    ```

    This ensures that when the NSE successfully starts a Live Activity (in background or foreground), the companion push notification is completely suppressed -- no banner, no sound, no badge increment.
