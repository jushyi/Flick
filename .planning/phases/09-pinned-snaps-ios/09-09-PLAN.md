<objective>
Create the pinnedNotificationService with notification dismissal on snap view and re-delivery when notifications are swiped away, then wire it into SnapViewer and App.js.

Purpose: This is the recipient-side logic for persistent pinned snap notifications. When a pinned snap is viewed, the notification must be dismissed. When a notification is swiped away but the snap is still unviewed, it must be re-delivered. This replaces the old Live Activity start/end lifecycle.

Output: pinnedNotificationService.js with dismissal + re-delivery, unit tests, SnapViewer wiring, App.js foreground check.
</objective>

    **Create src/services/pinnedNotificationService.js**

    This service manages pinned snap notification lifecycle on the recipient side. It follows the project's standard service pattern with `{ success, error }` returns where applicable, uses the logger utility (not console.log), and includes iOS Platform guards.

    **Imports:**
    ```javascript
    import { Platform } from 'react-native';
    import * as Notifications from 'expo-notifications';
    import AsyncStorage from '@react-native-async-storage/async-storage';
    import logger from '../utils/logger';
    ```

    **Function 1: `dismissPinnedSnapNotification(pinnedActivityId)`**
    - iOS Platform guard: return immediately on non-iOS or if pinnedActivityId is falsy
    - Call `Notifications.getPresentedNotificationsAsync()` to get all presented notifications
    - Filter for notifications where `request.content.data?.pinnedActivityId === pinnedActivityId`
    - Call `Notifications.dismissNotificationAsync(n.request.identifier)` for each match
    - Also clear the pinnedActivityId from the local tracking list (see trackPinnedSnap below)
    - Log success via logger.info with count of dismissed notifications
    - Wrap in try/catch, log errors via logger.warn, best-effort (do not throw)
    - This pattern matches the existing dismissal pattern in useConversation.js

    **Function 2: `trackPinnedSnap(pinnedActivityId, metadata)`**
    - Stores pinned snap metadata in AsyncStorage for re-delivery checks
    - Key: `pinned_snaps_active` (JSON array of objects)
    - Each object: `{ pinnedActivityId, conversationId, senderName, caption, pinnedThumbnailUrl, trackedAt }`
    - metadata parameter contains: `{ conversationId, senderName, caption, pinnedThumbnailUrl }`
    - Deduplicates by pinnedActivityId before appending
    - iOS Platform guard

    **Function 3: `clearPinnedSnap(pinnedActivityId)`**
    - Removes a pinned snap from the AsyncStorage tracking list
    - Called by dismissPinnedSnapNotification after successful dismissal
    - iOS Platform guard

    **Function 4: `checkAndRedeliverPinnedSnaps()`**
    - iOS Platform guard: return immediately on non-iOS
    - Read the local tracking list from AsyncStorage (`pinned_snaps_active`)
    - If empty, return early
    - Call `Notifications.getPresentedNotificationsAsync()` to get currently presented notifications
    - Build a Set of pinnedActivityIds from presented notifications' data payloads
    - For each tracked pinned snap not in the presented Set:
      - Schedule an immediate local notification via `Notifications.scheduleNotificationAsync`:
        ```javascript
        {
          content: {
            title: snap.senderName,
            body: snap.caption || '',
            data: {
              type: 'snap',
              conversationId: snap.conversationId,
              pinnedActivityId: snap.pinnedActivityId,
              pinned: 'true',
            },
          },
          trigger: null, // Deliver immediately
        }
        ```
      - Note: local notifications do NOT go through the NSE, so the re-delivered notification will not have the thumbnail image. This is acceptable per 09-RESEARCH.md Open Question 1 -- the user already saw the image on first delivery. Keep it simple.
    - Log the re-delivery count via logger.info
    - Wrap everything in try/catch, best-effort

    **Export all four functions as named exports.**

    - `react-native`: mock Platform.OS

    Test cases (minimum 10 tests):

    3. `dismissPinnedSnapNotification` finds and dismisses matching notification by pinnedActivityId in data
    4. `dismissPinnedSnapNotification` does not dismiss non-matching notifications
    5. `dismissPinnedSnapNotification` handles empty presented notifications list gracefully
    6. `dismissPinnedSnapNotification` catches and logs errors from getPresentedNotificationsAsync
    7. `trackPinnedSnap` stores metadata in AsyncStorage under pinned_snaps_active key
    8. `trackPinnedSnap` deduplicates by pinnedActivityId
    9. `clearPinnedSnap` removes the specified snap from tracking list
    10. `checkAndRedeliverPinnedSnaps` schedules local notification for tracked snaps missing from presented list
    11. `checkAndRedeliverPinnedSnaps` does NOT re-deliver snaps that are already presented
    12. `checkAndRedeliverPinnedSnaps` returns early on Android
    13. `checkAndRedeliverPinnedSnaps` handles empty tracking list gracefully

    Follow project test conventions: use `describe`/`it` blocks, mock setup in `beforeEach`, `jest.clearAllMocks` in setup.
