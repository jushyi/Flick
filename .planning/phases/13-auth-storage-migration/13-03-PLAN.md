**PowerSync local-only table schema** (add to `src/lib/powersync/schema.ts` or define inline):
```typescript
// Local-only table -- not synced to server
const UPLOAD_QUEUE_TABLE = 'upload_queue';
// Columns: id TEXT PK, user_id TEXT, media_uri TEXT, media_type TEXT DEFAULT 'photo',
//          duration REAL, status TEXT DEFAULT 'pending', attempts INTEGER DEFAULT 0,
//          created_at INTEGER, backend TEXT DEFAULT 'supabase', error_message TEXT,
//          photo_id TEXT, thumbnail TEXT
```

Since PowerSync may not be installed/configured yet (Phase 14), use a **fallback approach**: try to use PowerSync local table, fall back to an in-memory Map + AsyncStorage persistence under key `@supabase_upload_queue`. This makes the queue work NOW while PowerSync is set up in Phase 14.

**Key implementation details:**

1. `initializeQueue()`:
   - Check AsyncStorage for key `@upload_queue` (old Firebase queue)
   - If items exist with no `backend` field or `backend: 'firebase'`, process each via the OLD Firebase `storageService.uploadPhoto`/`uploadVideo` (import from `./firebase/storageService`)
   - After draining, clear `@upload_queue` key
   - Initialize new queue from AsyncStorage key `@supabase_upload_queue`
   - Set `isInitialized = true`

2. `addToQueue(userId, mediaUri, mediaType = 'photo', duration = null)`:
   - Generate UUID for item ID and photoId
   - Generate thumbnail via `generateThumbnail(mediaUri)` from Supabase storageService
   - Create queue item: `{ id, userId, mediaUri, mediaType, duration, status: 'pending', attempts: 0, createdAt: Date.now(), backend: 'supabase', photoId, thumbnail }`
   - Persist to AsyncStorage under `@supabase_upload_queue`
   - Trigger `processQueue()` (non-blocking)
   - Return `{ photoId, thumbnail }`

3. `processQueue()`:
   - If already processing, return (mutex via `isProcessing` flag)
   - Get all items from queue, filter `status === 'pending' || status === 'retry'`, sort by createdAt
   - For each item sequentially:
     - If `mediaType === 'photo'`: call `uploadPhoto(item.userId, item.photoId, item.mediaUri)`
     - If `mediaType === 'video'`: call `uploadVideo(item.userId, item.photoId, item.mediaUri)`
     - If `mediaType === 'snap'`: call `uploadSnapPhoto(item.userId, item.photoId, item.mediaUri)`
     - On success: update item status to `'completed'`, create Supabase photo record (insert into photos table with status 'developing', thumbnail, storage_path, etc.)
     - On failure: increment attempts, if attempts >= 3 set status to `'failed'`, else set status to `'retry'`, wait `2^attempts * 1000` ms
   - Persist updated queue to AsyncStorage
   - Set `isProcessing = false`

4. `getQueueLength()`: Return count of items where `status === 'pending' || status === 'retry'`

5. `clearFailedItems()`: Remove items with `status === 'failed'`, persist

   - Remove: `const unsubscribe = onAuthStateChanged(getAuth(), async (firebaseUser) => { ... })`
   - Add: `const { data: { subscription } } = supabase.auth.onAuthStateChange(async (event, session) => { ... })`
   - Map events: `SIGNED_IN` and `INITIAL_SESSION` -> fetch user profile from Supabase `users` table, set user + userProfile state. `SIGNED_OUT` -> clear state. `TOKEN_REFRESHED` -> no-op.

3. **Add silent migration bridge** (runs once on mount, before auth listener):
   ```javascript
   // Check for Firebase token to migrate
   useEffect(() => {
     const attemptMigration = async () => {
       try {
         const firebaseAuth = getAuth();
         const firebaseUser = firebaseAuth.currentUser;
         if (firebaseUser) {
           const firebaseToken = await firebaseUser.getIdToken();
           const { data, error } = await supabase.functions.invoke('migrate-firebase-auth', {
             body: { firebaseToken },
           });
           if (data?.requiresVerification) {
             // Migration succeeded but needs OTP verification
             // Navigate to PhoneInputScreen with migration context
             setMigrationPhone(data.phone);
           }
           if (data?.success && !data?.requiresVerification) {
             // Fully migrated -- Supabase auth listener will pick up the session
             logger.info('Silent migration completed', { migrated: data.migrated });
           }
           // Sign out of Firebase after successful migration
           await firebaseAuth.signOut();
         }
       } catch (error) {
         logger.warn('Migration attempt failed, user will re-verify', { error: error.message });
       }
     };
     attemptMigration();
   }, []);
   ```
