**Step 3: Implement (src/services/uploadQueueService.ts):**

Import `supabase` from `@/lib/supabase`, storage functions from `./supabase/storageService`, logger from `../utils/logger`. Import AsyncStorage from `@react-native-async-storage/async-storage` (for drain only).

The service interfaces with PowerSync's local SQLite database for persistence. Use the PowerSync `db` handle (from a PowerSync provider or singleton) to execute SQL against the `upload_queue` table:
- `db.execute('INSERT INTO upload_queue ...')` for addToQueue
- `db.getAll('SELECT * FROM upload_queue WHERE status IN (?, ?) ORDER BY created_at', ['pending', 'retry'])` for processQueue
- `db.execute('UPDATE upload_queue SET status = ? WHERE id = ?', ['completed', id])` for marking done
- `db.execute('DELETE FROM upload_queue WHERE status = ?', ['failed'])` for clearFailedItems
- `db.get('SELECT COUNT(*) as count FROM upload_queue WHERE status IN (?, ?)', ['pending', 'retry'])` for getQueueLength

**Key implementation details:**

1. `initializeQueue()`:
   - Check AsyncStorage for key `@upload_queue` (old Firebase queue)
   - If items exist with no `backend` field or `backend: 'firebase'`, process each via the OLD Firebase `storageService.uploadPhoto`/`uploadVideo` (import from `./firebase/storageService`)
   - After draining, clear `@upload_queue` key from AsyncStorage
   - Log: `logger.info('Upload queue initialized', { drainedCount })`

2. `addToQueue(userId, mediaUri, mediaType = 'photo', duration = null)`:
   - Generate UUID for item ID and photoId
   - Generate thumbnail via `generateThumbnail(mediaUri)` from Supabase storageService
   - Insert into PowerSync upload_queue table
   - Trigger `processQueue()` (non-blocking)
   - Return `{ photoId, thumbnail }`

3. `processQueue()`:
   - If already processing, return (mutex via `isProcessing` flag)
   - Query PowerSync for pending/retry items ordered by created_at
   - For each item sequentially:
     - If `mediaType === 'photo'`: call `uploadPhoto(item.userId, item.photoId, item.mediaUri)`
     - If `mediaType === 'video'`: call `uploadVideo(item.userId, item.photoId, item.mediaUri)`
     - If `mediaType === 'snap'`: call `uploadSnapPhoto(item.userId, item.photoId, item.mediaUri)`
     - On success: update status to 'completed', create Supabase photo record
     - On failure: increment attempts, if attempts >= 3 set status 'failed', else 'retry', wait `2^attempts * 1000` ms

4. `getQueueLength()`: SQL COUNT of pending+retry items

5. `clearFailedItems()`: SQL DELETE of failed items

3. **Add silent migration bridge** (runs once on mount):
   ```javascript
   useEffect(() => {
     const attemptMigration = async () => {
       try {
         const firebaseAuth = getAuth();
         const firebaseUser = firebaseAuth.currentUser;
         if (firebaseUser) {
           const firebaseToken = await firebaseUser.getIdToken();
           const { data, error } = await supabase.functions.invoke('migrate-firebase-auth', {
             body: { firebaseToken },
           });
           if (error) {
             logger.warn('Migration failed, user will re-verify', { error: error.message });
             // User will see login screen -- not silent, but safe fallback
           } else if (data?.access_token && data?.refresh_token) {
             // CRITICAL: Set the Supabase session with real tokens from Edge Function
             await supabase.auth.setSession({
               access_token: data.access_token,
               refresh_token: data.refresh_token,
             });
             // onAuthStateChange will fire SIGNED_IN -> AuthContext picks up session
             logger.info('Silent migration completed', { migrated: data.migrated });
           }
           // Sign out of Firebase after migration attempt (success or failure)
           await firebaseAuth.signOut();
         }
       } catch (error) {
         logger.warn('Migration attempt failed', { error: error.message });
       }
     };
     attemptMigration();
   }, []);
   ```
