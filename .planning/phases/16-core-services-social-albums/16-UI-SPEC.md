# Phase 16 -- UI Design Contract

> Visual and interaction contract for the social and albums service rewrite. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Nature: Backend Service Rewrite

Phase 16 is a **backend-only rewrite** -- replacing six Firebase services with Supabase equivalents. All screens (FriendsList, Activity, BlockedUsers, ReportUser, ContactsSync, CreateAlbum, AlbumGrid, MonthlyAlbumGrid, PhotoDetail comment section) already exist and must remain **visually identical**. This contract documents the existing visual system as a preservation target, not a design exercise.

No new screens, layouts, or components are introduced. The UI contract exists to ensure the service swap does not regress any visual or interaction behavior.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, list item padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, header spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (empty state spacing) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions: Touch targets use 44px minimum (iOS HIG) for all interactive elements (friend request accept/decline buttons, block/unblock actions, comment like buttons).

## Typography

Existing system -- preserved exactly. Pixel fonts render visually larger than standard fonts, so sizes are reduced. 4 distinct sizes, 2 weights.

| Role | Size | Font Family | Line Height |
|------|------|-------------|-------------|
| Caption / Label / Button | 12px | Silkscreen_700Bold (buttons), SpaceMono_400Regular (captions, timestamps, meta text) | 16px (1.33) |
| Body / Input | 14px | SpaceMono_400Regular | 20px (1.43) |
| Subheading | 16px | Silkscreen_400Regular | 22px (1.375) |
| Title | 18px | PressStart2P_400Regular | 24px (1.33) |

Font weights (2 total): Regular (400), Bold (700).

Role mappings within the 4-size system:
- Input text: 14px SpaceMono_400Regular (comment input, album name input) -- uses Body size
- @mention highlight: 14px SpaceMono_700Bold (bold variant of Body for mentioned usernames)
- Timestamps and meta text: 12px SpaceMono_400Regular -- uses Caption/Label size

## Color

Existing CRT-inspired dark theme -- preserved exactly.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0A0A1A | All screen backgrounds (CRT navy-black) |
| Secondary (30%) | #161628 | Cards, friend list items, comment bubbles, album covers |
| Accent (10%) | #00D4FF | Primary CTA buttons, active friend request accept, interactive highlights |
| Destructive | #FF3333 | Block, remove friend, delete album, report submission |

Accent (#00D4FF electric cyan) reserved for:
- "Add Friend" / "Accept" button backgrounds
- Active toggle states
- Comment like heart when liked
- Album creation CTA
- Contact sync "Add" buttons

Secondary accent (#FF2D78 hot magenta) used for:
- Unviewed story ring gradients (not directly in this phase)

Tertiary surface (#252540) used for:
- Pill/chip backgrounds (mention chips, contact result cards)
- Secondary button backgrounds (decline, cancel)

## Interaction Contracts

These interactions must remain functionally identical after the service swap.

### Friendship Operations
| Action | Current Behavior | New Backend | UI Expectation |
|--------|-----------------|-------------|----------------|
| Send request | Instant UI update, Firestore write | PowerSync local write | Same instant feel (local SQLite write is synchronous) |
| Accept request | Instant UI update, Firestore write | PowerSync local write | Same instant feel |
| Decline request | Item removed from list, Firestore delete | PowerSync local delete | Same instant removal |
| Unfriend | Alert confirmation, then remove | PowerSync local delete | Same confirmation dialog, same removal |
| Friend list load | Firestore query + user profile join | PowerSync local query + TanStack cache | Must render at same speed or faster |

### Comment Operations
| Action | Current Behavior | New Backend | UI Expectation |
|--------|-----------------|-------------|----------------|
| Load comments | Firestore onSnapshot real-time | TanStack useQuery + Supabase Realtime | Comments appear without manual refresh |
| Post comment | Optimistic add to list | TanStack mutation | Comment appears instantly in thread |
| Delete comment | Confirmation + remove from list | TanStack mutation | Same confirmation, same removal |
| @mention autocomplete | Query friends from Firestore | PowerSync local SQLite + TanStack cache | Suggestions appear instantly (offline-capable) |
| Like comment | Toggle heart, increment count | TanStack mutation | Same toggle animation, same count update |

### Album Operations
| Action | Current Behavior | New Backend | UI Expectation |
|--------|-----------------|-------------|----------------|
| Create album | Navigate to CreateAlbum, save | TanStack mutation | Same navigation flow |
| Add photos | Photo picker, array update | TanStack mutation (junction table insert) | Same picker, optimistic grid update |
| Remove photo | Remove from album | TanStack mutation (junction table delete) | Same removal animation |
| Delete album | Confirmation + navigate back | TanStack mutation | Same confirmation dialog |
| Monthly albums | Client-side grouping | Supabase RPC server-side grouping | Same grid layout, same month headers |

### Block/Report Operations
| Action | Current Behavior | New Backend | UI Expectation |
|--------|-----------------|-------------|----------------|
| Block user | Confirmation + blocked user disappears | TanStack mutation + RLS enforcement | Same confirmation, blocked content disappears via RLS |
| Unblock user | Immediate unblock, remove from list | TanStack mutation | Same immediate removal from blocked list |
| Report user | Navigate to ReportUser screen | TanStack mutation | Same navigation, same form |

### Contact Sync Operations
| Action | Current Behavior | New Backend | UI Expectation |
|--------|-----------------|-------------|----------------|
| Sync contacts | Permission + batch Firestore queries | Permission + single Supabase RPC | Same permission flow, faster results |
| Add from contacts | Send friend request | PowerSync local write | Same instant request sent |

## Loading & Error States

All existing loading/error patterns must be preserved:

| State | Current Implementation | Preservation Rule |
|-------|----------------------|-------------------|
| List loading | PixelSpinner component (centered) | Keep PixelSpinner, do not replace with skeleton (skeletons are Phase 19 scope) |
| Action loading | Per-item loading state (actionLoading === item.id) | Keep per-item loading indicators on friendship/block actions |
| Error display | Alert.alert() with error message | Keep Alert.alert pattern -- new services throw errors, TanStack onError catches and displays |
| Empty lists | ListEmptyComponent on FlatList | Keep existing empty state components verbatim |
