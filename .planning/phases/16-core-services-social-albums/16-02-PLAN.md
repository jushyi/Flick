Import `supabase` from `@/lib/supabase`. Import `logger` from `@/utils/logger`.

Define types:
```typescript
export interface Comment {
  id: string;
  photoId: string;
  userId: string;
  parentId: string | null;
  mentionedCommentId: string | null;
  text: string;
  mentions: string[];
  mediaUrl: string | null;
  mediaType: string | null;
  likeCount: number;
  createdAt: string;
  user?: {
    username: string;
    displayName: string;
    profilePhotoPath: string | null;
  };
}

interface AddCommentParams {
  photoId: string;
  userId: string;
  text: string;
  parentId?: string | null;
  mentionedCommentId?: string | null;
  mentions?: string[];
  mediaUrl?: string | null;
  mediaType?: string | null;
}
```

Constants: `MAX_COMMENT_LENGTH = 2000`, `MAX_MENTIONS_PER_COMMENT = 10`.

1. `getComments(photoId: string): Promise<Comment[]>` -- Query: `supabase.from('comments').select('*, user:users(username, display_name, profile_photo_path)').eq('photo_id', photoId).order('created_at', { ascending: true })`. Map snake_case to camelCase in return.

2. `addComment(params: AddCommentParams): Promise<Comment>` -- Validate text length <= MAX_COMMENT_LENGTH, validate mentions.length <= MAX_MENTIONS_PER_COMMENT. **Flat threading:** If `parentId` is provided, query that comment to check if IT has a parent_id. If yes, use the parent's parent_id as this comment's parent_id (flatten to top-level). Set mentioned_comment_id to the original parentId value. Insert via `supabase.from('comments').insert({...}).select('*, user:users(username, display_name, profile_photo_path)').single()`. Map and return.

3. `deleteComment(commentId: string): Promise<void>` -- `supabase.from('comments').delete().eq('id', commentId)`. Throw on error.

4. `likeComment(commentId: string, userId: string): Promise<void>` -- `supabase.from('comment_likes').insert({ comment_id: commentId, user_id: userId })`. Throw on error.

5. `unlikeComment(commentId: string, userId: string): Promise<void>` -- `supabase.from('comment_likes').delete().eq('comment_id', commentId).eq('user_id', userId)`. Throw on error.

6. `getCommentLikes(commentId: string): Promise<string[]>` -- `supabase.from('comment_likes').select('user_id').eq('comment_id', commentId)`. Return array of user_id strings.

Helper function `mapComment(row: any): Comment` to convert snake_case DB row to camelCase Comment type.

**Create src/hooks/useComments.ts:**

```typescript
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import { useEffect } from 'react';

import { supabase } from '@/lib/supabase';
import { queryKeys } from '@/lib/queryKeys';
import * as commentService from '@/services/supabase/commentService';

import logger from '@/utils/logger';
```

Export these hooks:

1. `useComments(photoId: string)` -- Returns TanStack useQuery result.
   - queryKey: `queryKeys.comments.list(photoId)`
   - queryFn: `() => commentService.getComments(photoId)`
   - enabled: `!!photoId`
   - Inside useEffect: subscribe to Supabase Realtime channel `comments:${photoId}` listening for `postgres_changes` on `public.comments` table filtered by `photo_id=eq.${photoId}`. On any event (`*`), call `queryClient.invalidateQueries({ queryKey: queryKeys.comments.list(photoId) })`. Return cleanup: `supabase.removeChannel(channel)`. Dependencies: `[photoId, queryClient]`.

2. `useAddComment()` -- Returns TanStack useMutation.
   - mutationFn: `(params: AddCommentParams) => commentService.addComment(params)`
   - onSuccess: invalidate `queryKeys.comments.list(photoId)` where photoId comes from the params
   - onError: `logger.error('Failed to add comment', ...)`

3. `useDeleteComment()` -- Returns TanStack useMutation.
   - mutationFn: `({ commentId, photoId }: { commentId: string; photoId: string }) => commentService.deleteComment(commentId)`
   - onSuccess: invalidate `queryKeys.comments.list(photoId)`

4. `useLikeComment()` -- Returns TanStack useMutation.
   - mutationFn: `({ commentId, userId }: { commentId: string; userId: string }) => commentService.likeComment(commentId, userId)`
   - Optimistic update: increment like_count in cached comment data
   - onError: rollback
   - onSettled: invalidate `queryKeys.comments.list(photoId)` (photoId passed in mutation variables)

5. `useUnlikeComment()` -- Similar to useLikeComment but decrement.

**Create src/hooks/useMentionSuggestions.ts:**

The `users` table is NOT in PowerSync. Strategy: query friendships from PowerSync to get friend IDs, then look up cached user profiles from TanStack Query cache.

```typescript
import { useState, useEffect, useMemo } from 'react';
import { useQueryClient } from '@tanstack/react-query';

import { powerSyncDb } from '@/lib/powersync/database';
import { queryKeys } from '@/lib/queryKeys';

import logger from '@/utils/logger';
```

Export `useMentionSuggestions(currentUserId: string, searchText: string)`:

1. Get friend IDs from PowerSync: `powerSyncDb.getAll('SELECT user1_id, user2_id FROM friendships WHERE status = ? AND (user1_id = ? OR user2_id = ?)', ['accepted', currentUserId, currentUserId])`. Map to extract the OTHER user's ID.

2. For each friend ID, look up profile from TanStack cache: `queryClient.getQueryData(queryKeys.profile.detail(friendId))`. This gets cached profile data without network calls.

3. Filter by searchText: match username or display_name starting with searchText (case-insensitive).

4. Return array of `{ id, username, displayName, profilePhotoPath }`.

Use `useState` + `useEffect` for the async PowerSync query. Memoize the filtered results.
